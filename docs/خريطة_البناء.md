# خريطة البناء — نظام إدارة الاستثمار في نينوى

> المرجع: `وصف_النظام_الشامل.md`. ابنِ **مرحلةً مرحلة**؛ لا تنتقل قبل اجتياز **معايير القبول + الاختبارات + التزام Git**. خطّة أولاً لكل مرحلة.

## م0 — الأساس والقاعدة
- سقالة Next.js + TS strict + Tailwind + shadcn/ui + Framer Motion (RTL) + Readex Pro.
- مشروع Supabase: تفعيل PostGIS + pgvector. **مخطّط القاعدة من §ج.8** (الكيانات التسعة + العلاقات).
- سكربتات استيراد `/data/`: 491 شركة · 27 فرصة · 146 رخصة · 125 سجلاً قانونياً.
- المصادقة (مستخدم واحد) + `.env` (Anthropic · Supabase · MapTiler).
- **قبول:** القاعدة منشأة، البيانات مستوردة بالأعداد الصحيحة، الدخول يعمل، النشر على Render حيّ.

## م1 — أساس الخريطة
- MapLibre + MapTiler · حدود نينوى (maxBounds) + حدود زوم · النمط الكحلي المخصّص.
- طبقة الحدود (محافظة/أقضية/نواحٍ) من GeoJSON.
- **قبول:** الخريطة تفتح على نينوى، التنقّل مقيّد، النمط مطابق لـ§هـ.3/§هـ.4.

## م2 — الطبقات وأقسام السايدبار
- الطبقة الهرمية (قضاء/ناحية/حي) + المقاييس الثلاثة + التسميات.
- عرض القطع بالتلوين بالحالة (٥ حالات) + الطبقة المحرَّرة + أداة الرسم/التعريف.
- أقسام السايدبار: الفرص · الرخص (+سجلّ الزيارات) · الشركات · مكتبة المعايير · التقارير · تصميم فرصة · الإعدادات — CRUD + بحث/فلترة + تصدير.
- المبادئ: الوصول المزدوج · التصنيف بالحالة · الترابط (مصدر واحد + Realtime).
- **قبول:** كل قسم يعمل CRUD، التعديل ينعكس فوراً في الخريطة وكل قسم، الرسم ينشئ قطعة مفترضة.

## م3 — نافذة القطعة والإجراءات

- التابات الأربعة: (1) **الضوابط الحتمية §ج.9 — دوالّ مختبَرة**؛ (2) التوصيات الذكية؛ (3) إنشاء معايير؛ (4) التقرير.
- **قبول:** تاب الضوابط يطابق §ج.9 باستشهاد، ويجتاز **اختبار وحدة لكل ضابط/معيار**؛ التكييف بالحالات الخمس صحيح.

## م4 — القانون والذكاء (RAG)
- إدخال الطبقة القانونية (Akoma Ntoso §ج.5) + بناء التضمينات (سكربت Python · pgvector/HNSW).
- المستشار القانوني بشقّيه (استمارة + أسئلة حرّة) — **مصدر الطبقة القانونية حصراً**، باستشهاد.
- التوصيات/المعايير (RAG + ويب وفق الإعدادات) — مؤشَّرة كاجتهاد.
- **قبول:** المستشار يجيب باستشهاد دقيق بلا تأليف؛ خارج النطاق يقول «غير متوفّر».

## م5 — الهيدبار والبحث والتقارير والإعدادات
- الهيدبار: عدّادات لحظية **حتمية** + **البحث الفائق** (لغة طبيعية · عناصرنا أولاً ثم كل المعروض · ضمن نينوى).
- التقارير الذكية (لوحات/تشارتات/فلاتر/تصدير — حتمي).
- الإعدادات: الحساب · العرض · **الذكاء والويب (اختيار النموذج + إدارة مفاتيح API)** · التصدير.
- **قبول:** العدّادات من بيانات حقيقية، البحث يجد أبسط عنصر، الإعدادات تضبط النموذج/المفاتيح فعلياً.
