# موجز معرفي: كيانات الأعمال وطبقة العرض المركزية

## الغرض
هذا المجال يغطّي إدارة كيانات الأعمال الخمسة (الفرص الاستثمارية · الرخص · الشركات · مكتبة المعايير · التقارير، مع الزيارات الميدانية المرتبطة بالرخص) و**طبقة العرض المركزية** التي تفرض قواعد المخرجات §ح على كل ما يُعرَض أو يُصدَّر. الهدف: CRUD نظيف موحّد + إخراج ملتزم حرفياً بـ«صفر تأليف»، أرقام لاتينية، منع المعرّف الداخلي، ومنع كشف التحقّق الداخلي.

## البنية وتدفّق البيانات
- **مصدر واحد + تزامن لحظي:** كل طفرة عبر Server Action (`'use server'`) ← Postgres ← Realtime ← إبطال ذاكرة TanStack ← انعكاس فوري في كل الأقسام والخريطة. القلب: `src/lib/data/realtime.ts` (`useRealtimeSync`) يشترك في `postgres_changes` لـ13 جدولاً ويُبطل بدقّة `['table',name]`، و`['counts']` (لجداول `COUNTED_TABLES` الستة)، و`['dashboard_stats']` (لجداول `STATS_TABLES` الأربعة)، و`['map_parcels']`+`['table','field_options']` (لجداول `PARCEL_TABLES`)، بالإضافة لمفاتيح مخصّصة (`insights`/`map_elements_geo`/`parcel_photos`/`parcel_models`) حسب الجدول.
- **القراءة:** `useTable<T>(table)` (`lib/data/use-table.ts`) بمفتاح `['table',name]`، يجلب **كل** الصفوف بالترقيم (`PAGE=1000`) بترتيب مفتاح أساسي مستقر (خريطة `PK`؛ الافتراضي `id`) لتفادي تكرار/فقدان صفوف PostgREST. مرافقوه: `use-counts` (عدّ لحظي `count exact head`)، `use-field-options` (القاموس الموحّد للمنسدلات من **view** `field_value_options`)، `use-dashboard-stats` (RPC حتمي `dashboard_stats`).
- **الطفرات:** كل feature له `actions.ts` بـ `ActionResult = {ok:true}|{ok:false;error}`. الفرص/الرخص تستخدم **فصل صريح** update vs insert (الرخص تتجنّب `upsert` لئلا يفجّر NOT NULL — خطأ 23502؛ الشركات تستخدم `upsert` على `id`). المعرّفات: الفرص/الرخص `record_id=Date.now()` + `kind`؛ الشركات تولّد المعرّف بـ`crypto.randomUUID()` (حين لا يُمرَّر `id`) ثم `upsert`؛ المعايير تستخدم `insert(values)` صرفاً عند الإنشاء **دون توليد UUID في العميل** (تعتمد افتراضي القاعدة).
- **النماذج:** تحوّل FormData إلى قيم مع **فراغ→null** (لا تأليف)، تُبطل الذاكرة بعد النجاح، وتُظهر toast بسبب واضح عند الفشل مع إبقاء النموذج (§ز.2). تستخدم `OptionField`/`ComboField` لحقول الخيارات: نموذج الفرص المرجعي (`opportunity-form.tsx`) يستخدم `OptionField` **فقط**، بينما `ComboField` يظهر في نماذج الشركات/الرخص/المعايير (المرجعيّة هنا على مستوى المجال لا على نموذج الفرص بعينه).
- **اللوحات:** بنية موحّدة (نموذج الفرص = المرجع): `useTable` + فلاتر صامدة عبر `usePersistentState`/`usePersistentScroll` + تحديد متعدّد + طيّ بطاقات + تصدير وفق `settings.default_export`. العارض (`isViewer`) محجوب عن الإضافة/التعديل/الحذف/الرسم مع إبقاء العرض/التصدير/PDF.

## طبقة العرض المركزية (§ح)
- `src/lib/format.ts` — البذرة: `toLatinDigits` (يحوّل العربية-الهندية U+0660-0669 والفارسية U+06F0-06F9 دفاعياً)، `formatNumber` (`Intl.NumberFormat('en-US')` لضمان اللاتينية + فواصل آلاف)، الثابت `NOT_AVAILABLE='غير متوفّر'`.
- `src/lib/display.ts` — المساعدات: `orNA` (القيمة أو «غير متوفّر» + توحيد لاتيني)، `formatDate` (YYYY-MM-DD مقتطعة)، `formatArea` (`formatNumber` + «م²»).
- **كل** قيمة معروضة في اللوحات/التفاصيل تمرّ عبر هذه الدوالّ حصراً (مثبَّت في كل ملفات الواجهة المقروءة). الاختبارات: `format.test.ts` يثبّت اللاتينية والتحويل (عربي-هندي وفارسي) وثبات «غير متوفّر».

## التصدير CSV/PDF
- `lib/export-csv.ts`: CSV من المتصفّح، تهريب الخلايا (تضعيف `"`)، BOM للعربية في Excel، `toLatinDigits` لكل خلية. واجهة `CsvColumn {key,label,format?}`.
- `lib/export-table.ts`: موحّد — إن كانت الصيغة `pdf` يرسل POST إلى `/api/pdf/table` (مع رسم اختياري)؛ غير ذلك CSV محلي. يُعيد `boolean` (false عند فشل توليد PDF → toast).
- مسار `src/app/api/pdf/table/route.ts`: يتحقّق من الجلسة (401 بلا كشف)، **يقصّ** المدخلات (30 عمود/3000 صف/أطوال)، يبني غلافاً برانداً، يصيّر.
- `lib/pdf/render.ts`: Puppeteer **singleton** + خطّ Readex مضمّن base64 (تراجُع Google عند الغياب) + `esc()` للتهريب + STYLES كاملة + ترويسة/تذييل وترقيم وهامش A4 RTL.
- البناة: `table-report.ts` (`coverHtml` غلاف · `chartSvg` رسم أعمدة SVG حتمي لاتيني · `tableReportBody`)، `consultation-report.ts` (`answerToHtml` تحويل ماركداون **حرفياً** + تمييز الاستشهادات كرقائق `.cite` عبر regex، بلا محتوى جديد)، `branding.ts` (هوية من settings). الاختبارات `pdf-builders.test.ts` تثبّت التهريب، الخلية الفارغة (لا تأليف)، رقاقة الاستشهاد، العنوان الافتراضي.

## تمييز الإلزامي عن الاجتهاد + الذكاء
- إثراء الشركات (`companies/enrich-actions.ts`) معلَّم صراحةً **🟩** (رأي غير مُلزِم): بحث ويب خادمي عبر Anthropic، يُرجِع JSON **بمصدر فقط** (يرفض أي مُقترَح بلا مصدر أو رقم غير صرف)، ولا يُحفَظ شيء إلا **باعتماد المستخدم** (`applyEnrichment`)، ويوثّق المصدر في `source[]`. النموذج وبحث الويب من الإعدادات. هذا التطبيق الوحيد للذكاء في المجال.

## المنطق الحتمي (ليس ذكاءً)
- `reports/report-aggregations.ts`: دوالّ نقية (`normalize`/`applyFilters`/`totals`/`bySector`/`byDistrict`/`byState`/`byYear`) مع **منع العدّ المزدوج للمساحات** (فرصة على نفس قطعة رخصة ← مساحتها 0). مُختبَرة في `report-aggregations.test.ts`.
- `licenses/fields.ts::normalizeLicenseStatusForSave`: دالّة نقية مُختبَرة (إنشاء بلا حالة→`in-progress`؛ تحديث بلا حالة→حذف المفتاح فلا تُمسّ المخزّنة — إصلاح انقلاب الحالة). الاختبار في `licenses/fields.test.ts`.
- `companies/fields.ts::isEligible`: غير مستثناة ∧ `meets_250k_threshold===true`.

## الثوابت غير القابلة للتفاوض (المرتبطة بمجالي)
الأرقام لاتينية دائماً · «غير متوفّر» لا اختلاق · منع المعرّف الداخلي وكشف التحقّق · تمييز 🟦/🟩 · مفاتيح API خادمية فقط · المنطق القانوني/الحتمي دوالّ نقية مع اختبارات · المصدر الواحد + التزامن · توحيد التصنيفات (رمز للتخزين/عربي للعرض). نزاهة الأهلية: `is_excluded` لا تُقلب صامتاً عند حفظ جزئي. العارض (`isViewer`) محجوب عن CRUD.

## الحالة الراهنة
**مبنيّ ويعمل (حيّ على Render):** CRUD كامل للكيانات الخمسة، طبقة عرض مُختبَرة، تصدير CSV/PDF براندد، خطافات بيانات + تزامن لحظي، تقارير حتمية + تقارير/سجلّ زيارات بصور، إثراء شركات 🟩. **المؤجَّل:** ربط الرخص بالشركات/الفرص (م3)، تحرير مصفوفات الشركة والإثراء الكامل (م4)، سجلّ المشاريع، `updated_at_label`.

## إرشاد البناء
انسخ نموذج الفرص: `fields.ts` (بلا معرّف داخلي/تحقّق) → `actions.ts` (ActionResult + فصل update/insert + توحيد التصنيفات) → نموذج (فراغ→null + إبطال ذاكرة + toast) → لوحة (`useTable` + فلاتر + تصدير). تذكّر أنّ نموذج الفرص نفسه يستخدم `OptionField` فقط (الشركات/الرخص/المعايير تضيف `ComboField`). مرّر **كل** قيمة معروضة عبر `orNA`/`formatDate`/`formatArea`/`formatNumber`. أي منطق قرار = دالّة نقية + اختبار vitest، لا ذكاء. أي نداء ذكاء = `'use server'` بنموذج الإعدادات. عند جدول جديد: حدّث `PK` (use-table) و`TABLES`/المجموعات (realtime). للتصدير مرّر القيم منسَّقة مسبقاً (المسار يقصّ ويهرّب فقط). احجب CRUD عن `isViewer`. عند الغموض: ارجع للوصف ثم اسأل — لا تخمّن.

## الأسئلة المفتوحة / المخاطر
محرّك الربط (company_ref/opportunity_ref) غير مبنيّ — يظهر أثره في `opportunities-panel.tsx` عبر `isAvailable` المعتمد على `license_ref` · تحرير مصفوفات الشركة مؤجَّل · `updated_at_label` غير محسوم · علاقة `isEligible` بمحرّك الضوابط §ج.9 الكامل غير مُعرَّفة هنا · حدّ تصدير 3000 صف قد يحتاج مراجعة مع نموّ البيانات.
