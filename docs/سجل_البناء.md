- **القشرة (§3/§4/§7):** هيدبار الجوال = صورة أكبر + عنوان أوضح بلا زرّ بحث + إزاحة `--sat`. **دوك عائم** على الحافة اليمنى الفعلية (`right-2`، RTL) بنفس `SECTIONS` المفلترة بالأدوار + خروج، يعلو الورقة عند فتحها. شريط KPI أفقي رفيع تحت الهيدبار (يُعيد استخدام `CHIPS/NUM_GRADIENT`). توجيه الأقسام: الفرص/الرخص ← **ورقة سفلية**؛ البقية ← **ملء شاشة** (نفس مكوّنات اللوحات بلا تغيير منطقي).
- **الخريطة (§5):** عرض كامل على الجوال (`right-0 md:right-20`). تأطير المحافظة وكل طيران عبر **camera padding** (لا تغيير حجم canvas) يحترم شريط KPI والورقة الحيّة وشريط البحث؛ أزرار زوم MapLibre زجاجية مكبّرة مرفوعة فوق شريط البحث. الحدود من `bbox(nineveh_governorate.geojson)` القائم (لا تأليف).
- **الورقة السفلية (§6):** التقام «نصف/موسّع» بسحب `framer-motion` (transform فقط). **اللحظة البصرية:** نقر زر الموقع في البطاقة ← طيران سينمائي + الورقة إلى «نصف» + تطفو بطاقة صور القطعة (جوال فقط) + القائمة مفتوحة. مصدر واحد لارتفاع الورقة الحيّ (`mobile-sheet-store`) يستهلكه camera padding + قصّ البطاقة + إزاحة الدوك. شريط البحث السفلي (§8) يُخفى تحت الورقة.
- **بطاقة الصور (§9):** أبعاد مرنة على الجوال (عرض يتّسع للشاشة + ارتفاع محجوز ضمن الباند المرئي فوق الورقة، صورة `flex-1`)؛ الديسكتوب يبقى 340×446 حرفياً؛ الخط من **مركز القطعة** (centroid قائم).
- **ملفات جديدة:** `manifest.ts` · `components/{install-hint,mobile-search-bar}.tsx` · `shell/{mobile-dock,mobile-section-sheet,mobile-sheet-store}.ts(x)`. **معدّلة:** `globals.css · layout · page · headbar · mobile-kpis · app-sidebar · investment-map · selected-parcel-card`.
- **مراجعة تخاصمية متعددة الوكلاء:** أُصلحت ملاحظة md+ حقيقية (حارس بطاقة القطعة كان بعرض الـcanvas → صار `matchMedia` viewport، يمنع تحوّل البطاقة لوضع الجوال في النطاق 768–847px) + تنظيفات (RTL safe-area للدوك، توجيه مكرّر، كتابة CSS var ميتة). الاختبارات 54/54 · typecheck/lint نظيفان · كونسول نظيف.

## م8.4 — صقل الجوال (حزمة طلب · جلسة 2026-06-15، متحقَّق حيّاً عند 440×956 + 1280px)
> نفس القيد (جوال فقط، صفر تغيير على md+).
- **زر «كامل نينوى»:** زر دائري بنصّه تحت الدوك (الوضع العمودي) يعيد العرض لكامل المحافظة عبر قناة `requestResetView`/`onResetView` الجديدة في `map-nav-store` (الاشتراك في `investment-map`). زر الريست في عنقود الأدوات صار `max-md:hidden` (يبقى للديسكتوب).
- **الدوك يتجانس مع الورقة:** عند فتح الفرص/الرخص يتحوّل الدوك إلى **شريط أفقي يعلو الورقة ويرتفع معها حيّاً** (bottom=ارتفاع الورقة) بدل الانزياح الرأسي الشاذّ.
- **الورقة 65%:** الانبثاق التلقائي = 65% من الشاشة (منطقة تمرير أوسع والخريطة أظهر)، والموسّع ≈90%.
- **KPI متحرّك (marquee):** الأرقام تمشي بحركة بطيئة سلسة لا نهائية (تدخل من اليمين) — **CSS خالص** (`@keyframes kpi-marquee`)، لا framer.
- **إصلاح حرج متحقَّق:** كان `motion.div` بحركة `repeat: Infinity` (نسخة framer من الـmarquee) **يوقف `AnimatePresence`** فلا تُغلَق أي لوحة قسم (الورقة وملء الشاشة على الجوال **ولوحة الديسكتوب الجانبية** أيضاً، لأن framer يحرّك حتى عنصراً `display:none`). التحويل إلى CSS أصلح الإغلاق على كل المسارات. متحقَّق: إغلاق الورقة/ملء الشاشة/الـaside يعمل، والريست يعيد لكامل نينوى، والديسكتوب سليم.
- **coverflow لكل البطاقات + تأكيد الطيران (حزمة طلب لاحقة):** البؤرة صارت **أعلى-وسط المنطقة المرئية** (40%) لا منتصف الحاوية (الحاوية أطول من المرئي بسبب transform)، مع حشوة علوية صغيرة تُبرز البطاقة الأولى وحشوة سفلية كبيرة (خارج الشاشة) تتيح الأخيرة بلوغ البؤرة، وإعادة حساب بعد استقرار حركة الفتح (380/780ms) — فكلّ البطاقات (الأولى والأخيرة ضمناً) تتمركز وتبرز عند تمريرها. **متحقَّق حيّاً E2E:** نقر زر الموقع على قطعة مرسومة (محطة تعبئة وقود السهل النموذجية) ← الورقة تُغلَق فوراً + الخريطة تطير للقطعة (50كم→100م) + تظهر تسميتها، بلا بطاقة صور تلقائية، بلا أخطاء كونسول. (القطع غير المرسومة: تُغلَق الورقة + تنبيه «لا حدود مرسومة» — لا طيران بلا إحداثيات، لا تأليف.) 54/54 اختبار · typecheck/lint/بناء نظيف · الديسكتوب سليم.

3. **تعريب تسميات قاعدة الخريطة**: ضمن الحزمة الثابتة (لا Google Maps — تغيير معماري كبير وشروط استخدامها تمنع عرض بلاطاتها خارج SDK الخاص بها): فحص أنماط/مصادر MapTiler الأغنى بـ`name:ar`، وتكثيف **طبقة التسميات المحرَّرة** (الحل السيادي §هـ.4 — ما يعرّفه المستخدم يظهر بالخريطة والبحث)، ومراجعة ترتيب coalesce للتسميات.

1. تجربة المستخدم المحلية الشاملة على الخادم الإنتاجي (هذه الجلسة جهّزته).
2. عند الرضا: اعتماد م6+م7 ← دمج `m7-enhancements` في `main` بوسمَي `m6-approved`/`m7-approved`.
3. م6.5: النشر على Render (Build: `npm install && npm run build` · Start: `npm run start` · Node 22 · المتغيّرات الستّة) + فحص دخاني — **بقرار المستخدم حصراً**.
4. فجوات موثّقة بانتظار قرار: إرسال الاستشارة (بريد/رابط) · تغيير اسم المستخدم · مدخل التوصيات من السايدبار (§هـ.4) · خريطة PDF بديلة (Static API محجوبة بالخطة).
