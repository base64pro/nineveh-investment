## ما نبنيه
تطبيق ويب **أحادي المستخدم** (المدير العام لهيئة استثمار نينوى)، محوره **خريطة تفاعلية** تربط القطع بالفرص والرخص والشركات والقانون. ثنائي الغاية: **قرار سيادي + ترويج للمستثمر**. الجودة المطلوبة: نظيف · رصين · ثابت · مبهر · **صفر خطأ**.

## الحزمة التقنية (ثابتة)
- **Frontend/full-stack:** Next.js (App Router) + TypeScript **strict** + Tailwind + shadcn/ui + Framer Motion — RTL
- **الخريطة:** MapLibre GL JS + MapTiler (متّجه) + deck.gl (طبقات WebGL)
- **القاعدة:** Supabase (PostgreSQL + PostGIS + pgvector/HNSW + Auth + Storage + Realtime)
- **الذكاء:** Anthropic API عبر **الخادم فقط** (Server Actions / API Routes) — RAG على pgvector
- **الحالة:** TanStack Query · **التصدير:** PDF خادمي (Puppeteer/react-pdf)
- **النشر:** Render (التطبيق) + Supabase (القاعدة)

## قواعد غير قابلة للتفاوض (صفر استثناء)
1. العربية + **RTL** في كل الواجهة. **الأرقام لاتينية دائماً (0123).**
2. **صفر تأليف:** كل رقم/ادّعاء من مصدر. الناقص يُعرَض **«غير متوفّر»** لا يُخترَع.
3. **الاستشهاد:** اسم القانون + سنة + مادة + بند. **يُمنع** عرض المعرّف الداخلي للسجل.
4. **لا كشف للتحقّق الداخلي** للمستخدم إطلاقاً.
5. **المنطق القانوني (الوصف §ج.9) = دوالّ نقية حتمية مع اختبارات وحدة — ليس ذكاءً.** الذكاء يصوغ الشرح فقط، لا يقرّر نتيجة قانونية.
6. **مفاتيح API لا تصل للعميل أبداً** — كل نداء ذكاء خادمي.
7. تمييز **🟦 إلزامي** (باستشهاد) عن **🟩 اجتهاد** (مؤشَّر «غير مُلزِم»).
8. **تحقّق مسبق من الرخص** قبل أي إجابة عن قطعة/فرصة.
9. النموذج المستخدم في **كل** وظائف الذكاء = **المختار في إعدادات المستخدم**.
